Field-level data inventory

All profile fields below are read only when already rendered on the current supported page. They are processed ephemerally on-device and are neither retained nor transmitted.

Visible field categoryWhy it is readStored or sent
Generic LinkedIn verification badgeExplains that details may be available; it gives no identity points by itselfNo
Structured government-ID, work-email, and education verification methods; account join date; verification date; contact/photo update recencyVerification, account-tenure, continuity, and small maintenance signals after a user-triggered scanNo; only categories and elapsed-day values derived from visible dates remain in route-local memory
Headline, About, Experience, Education, and Skills sectionsCompleteness, specificity, chronology, and cross-section consistencyNo
Visible employer links and role detailsClaimed-company affiliation evidenceNo
Visible account-age text, when presentAccount maturity evidenceNo
Visible activity, interaction, network counts, and recommendationsTime distribution, reciprocal engagement, network maturity, and social-proof evidenceNo
Whether a profile image is present or a default placeholder is renderedA narrowly gated thin/new-profile signal; image pixels and biometric traits are never analyzedNo

The extension does not read messages, cookies, authentication tokens, contact details, private APIs, hidden profile data, external websites, or facial/biometric attributes. It scrolls the current profile and opens the native “About this member” dialog only after the user presses “Click to verify authenticity.”
